RxSwift 特征序列:Single/Driver/Maybe/Completable

Posted on 2019-07-17

#1 Single

Single 类似 Observable ，但它不会发出一系列的元素，而是只发出一个 onSuccess(value) 或 onError(error)。 Single 比较常见的例子就是执行 HTTP 请求，然后返回一个应答或错误。不过我们也可以用 Single 来描述任何只有一个元素的序列。 Observable 的操作符基本都能在 Single 上使用。
Single 不会共享状态变化

#2 Driver

Driver 的目标是提供一种简便的方式在 UI 层编写响应式代码。
如果我们的序列满足如下特征，就可以使用它：
1.不会产生 error 事件
2.一定在主线程监听（MainScheduler）
3.共享状态变化（shareReplayLatestWhileConnected）

#3 Maybe

Maybe 是 Observable 的另外一个版本。它介于 Single 和 Completable 之间，它要么只能发出一个元素，要么产生一个 completed 事件，要么产生一个 error 事件。

发出一个元素或者一个 completed 事件或者一个 error 事件
不会共享附加作用
如果你遇到那种可能需要发出一个元素，又可能不需要发出时，就可以使用 Maybe。

#4 Completable

Completable 是 Observable 的另外一个版本。不像 Observable 可以发出多个元素，它要么只能产生一个 completed 事件，要么产生一个 error 事件。

发出零个元素
发出一个 completed 事件或者一个 error 事件
不会共享附加作用
Completable 适用于那种你只关心任务是否完成，而不需要在意任务返回值的情况。它和 Observable 有点相似。

#5 什么是共享状态变化

共享 附加作用：
Driver
Signal
ControlEvent

不共享 附加作用：
Single
Completable
Maybe

因此我们需要注意，如果一个网络请求序列，他不共享 附加作用，那每一次订阅时就会单独发起网络请求。这时最好改用 共享 附加作用 的序列，或者使用 share 操作符。

RxSwift Subject

Posted on 2019-06-30

Subject概括
Subject是一种桥接器或代理，在ReactiveX的某些实现中可用，既充当 Observer，又充当 Observable。因为它是一个 Observer，所以它可以订阅一个或多个 Observable，而且因为它是一个 Observable，所以它可以通过重新发出所观察到的项来传递这些项，它还可以发出新的项。

#1 AsyncSubject

简单来说 AsyncSubject 只能订阅到最后的 onNext(value) 而且要在 onCompleted 发出后, 如果发出 onError , AsyncSubject 就不能定义最后的 onNext(value)。

#2 BehaviorSubject

BehaviorSubject 会保存最后发送的值或是初始化时的值。当订阅 BehaviorSubject 在发送第一次的 onNext 元素时，Observable 首先是发送最近的元素（或默认值）。可通过 value() 方法获取最近的元素。

#3 PublishSubject

PublishSubject 只向 observer 发出订阅后的元素，订阅前的元素会忽略掉。

#4 ReplaySubject

ReplaySubject 向 oberver 发出 Observable 的所有项，而不管 Observable 何时订阅。 初始化时需要确定可以缓存元素数量。

RxSwift Combining

Posted on 2019-06-23 | Edited on 2019-06-24

#1 combineLatest

combineLatest 操作符的作用是合并多个 Observable， 只要其中一个 Observable 发出一个元素，合并后的 Observable 就会发出多个 Observable 最新的值。 前提是所有的 Observable 都发出个值，如果其中一条 Observable 没有发出过元素，合并后的 Observable 也不会有元素发出。

#2 concat

Concat 操作符将多个 Observable 的输出连接起来，使它们的行为类似于单个Observable，第一个 Observable 发出的所有元素都先于第二个Observable 发出的元素。若在第一个 Observable 没有发送 onCompleted 第二个 Observable 所发的元素是不会被接受到，只有在第一个 Observable 完成时，第二个 Observable 的元素才能被接收。

#3 Merge

将多个Observable 合并成一个，某个Observable 发送元素也会触发 Merge 后的 Observable。某一个 Observable 发出一个 onError 事件，那么被合并的 Observable 也会将它发出，并且立即终止序列。

#4 StartWith

将一些元素插入到 Observable 的头部。

#5 WithLatestFrom

#6 Zip

Zip方法返回一个 Observable，它将您选择由两个(或多个)其他 Observable 按顺序发出的元素的组合而成。而且它会等到每个 Observable 事件一一对应地凑齐之后再合并发送。

RxSwift Filtering

Posted on 2019-06-17

#1 Debounce

只有在特定的时间跨度已经过去而没有发出另一个元素，才从在新的Observable中发出一个元素

总结： 相邻两个元素的时间间隔少于 debounce 的时间就会被忽略掉，但会发送最后的元素该操作符确保没有两个元素在小于dueTime的时间内发出。

#2 DistinctUntilChanged

DistinctUntilChanged把相邻两个元素做判断，如果相等就忽略掉后一个的元素。也可以自定义过滤方法。

#3 ElementAt

只发出Observable中的第n(从0开始)个元素。

#4 Filter

Filter操作符只允许能通过提供的判断方法的元素，从而过滤Observable的元素。

#5 First

First操作符只发出这个Observabel对象发出的第一项，如果这个可观察对象完成时没有发出任何东西会返回nil，first会把源Observable转换成Single，Single以后再做介绍。

#6 IgnoreElements

跳过所有元素、完成或错误并在Observable发送完成(或错误)时结束。等价于filter，它总是返回false。

#7 Sample

sample 操作符将不定期的对源 Observable 进行取样操作。通过第二个 Observable 来控制取样时机。一旦第二个 Observable 发出一个元素，就从源 Observable 中取出最后产生的元素。

#8 Single

single 操作符将限制 Observable 只产生一个元素，与first类似。如果 Observable 只有一个元素，它将镜像这个 Observable 。如果 Observable 没有元素或者元素数量大于一，它将产生一个 error 事件。

#9 Skip

skip 操作符可以让你跳过 Observable 中头 n 个元素，只关注后面的元素。

#10 SkipDuration

skipUntil

跳过源 Observable 中头几个元素，直到另一个 Observable 发出一个元素。

skipWhile

skipWhile 操作符可以让你忽略源 Observable 中头几个元素，直到元素的判定为false后，它才镜像源 Observable。

#11 Take

take

通过 take 操作符源 Observable 只发出前 n 个元素。并且忽略掉后面的元素，直接结束序列。

take(duration)

take(duration) 只发出 n 秒前的元素。

takeLast

通过 takeLast 操作符你可以只发出尾部 n 个元素。并且忽略掉前面的元素。

takeUntil

忽略掉在第二个 Observable 产生事件后发出的那部分元素

takeWhile

生成一个 Observable 直到某个元素的判定为 false

#12 Throttle

Throttle 操作符确保没有两个元素在小于dueTime的时间内发出。

如果把 throttle的值变为0.9的话

Throttle和Debounce的区别

Throttle: the original function be called at most once per specified period.
Debounce: the original function be called after the caller stops calling the decorated function after a specified period.

RxSwift Transforming

Posted on 2019-06-09 | Edited on 2019-06-16

关于RxSwift的转换方法

#1 Buffer

使用指定的调度程序运行计时器，将Observable的每个元素投射到缓冲区中，该缓冲区在缓冲区已满或已经过一定时间时发出。

打个比喻，在现实世界中，当所有的座位都被占满时，渡轮离开码头的行为，或者在预定的出发时间，无论哪个事件首先发生。

注意，如果源Observable发出一个onError通知，Buffer将立即传递该通知，而不会首先发出它正在组装的缓冲区，即使该缓冲区包含源观察对象在发出错误通知之前发出的项。

#2 FlatMap

将一个可观察序列的每个元素投射到一个可观察序列中，并将得到的可观察序列合并成一个可观察序列。

#3 FlatMapFirst

功能与flatMap相同，不过flatMapFirst，合并后只接受第一个元素，其他元素被简单地忽略。

#4 FlatMapLatest

flatMapLatest 与 flatMap 的唯一区别是：flatMapLatest 只会接收最新的 value 事
件

flatMapLatest 操作符将源 Observable 的每一个元素应用一个转换方法，将他们转换成 Observables。一旦转换出一个新的 Observable，就只发出它的元素，旧的 Observables 的元素将被忽略掉。

#5 Map

通过对每个值应用一个函数来转换Obervable发出的值，然后返回新的Observable

#6 MapWithIndex

mapWithIndex 也只是在 map 基础上添加了索引参数。

#7 Scan

scan将一个输入值作为发出的第一个项，它还将源Observable的元素作为第二项一起反馈回函数，然后通过返回一个新的值作为其第一项发射。

#8 ToArray

将源Observable转换成数组的Observable.

RxSwift Creating

Posted on 2019-06-05 | Edited on 2019-06-09

关于创建Observable

#1 Create

您可以使用create操作符从零开始创建一个可观察对象，传递给这个操作符一个函数，该函数接受observer作为其参数，通过适当地调用observer的onNext、onError和onCompleted方法，将这个函数编写为一个可观察的函数。
需要注意的是一个结构良好的Observable必须调用一次observer的onCompleted或onError方法，并且此后不能尝试调用observer的任何其他方法。

#2 Empty/Never

Empty

创建一个不发出任何项但正常终止的Observable(调用onCompleted)

Never
创建一个不发出任何项且不终止的可观察对象

Empty/Never这些对于测试目的很有用，有时也可以与其他可观测值结合使用，或者作为期望其他可观测值作为参数的操作符的参数。

#3 From

作用将各种其他对象和数据类型转换为可观察的

#4 Interval

创建一个Observable，它发出一个按给定时间间隔间隔的整数序列
Interval操作符返回一个可观察到的值，该值发出一个无限的升序整数序列，在两次发射之间选择一个固定的时间间隔。

#5 Just

创建一个发出特定值的Observable
与From类型，但要注意From会将数组里的元素取出再发送出去，而Just只是简单的将数组或对象作为一个单独值发送出去。

#6 Range

创建一个Observable，该对象发出特定范围的序列整数

#7 Repeat

创建一个Observable，该对象多次发出特定的项

该操作符重复发送一个项。

#8 Timer

创建一个Observable，它在给定的延迟之后发出特定的项

#9 Of

此方法创建一个Observable，该实例具有可变数量的元素。
功能与From相同，From接受数组为元素，Of接受可变数量的元素
