RxSwift Transforming

Posted on 2019-06-09 | Edited on 2019-06-16

关于RxSwift的转换方法

#1 Buffer

使用指定的调度程序运行计时器，将Observable的每个元素投射到缓冲区中，该缓冲区在缓冲区已满或已经过一定时间时发出。

打个比喻，在现实世界中，当所有的座位都被占满时，渡轮离开码头的行为，或者在预定的出发时间，无论哪个事件首先发生。

注意，如果源Observable发出一个onError通知，Buffer将立即传递该通知，而不会首先发出它正在组装的缓冲区，即使该缓冲区包含源观察对象在发出错误通知之前发出的项。

#2 FlatMap

将一个可观察序列的每个元素投射到一个可观察序列中，并将得到的可观察序列合并成一个可观察序列。

#3 FlatMapFirst

功能与flatMap相同，不过flatMapFirst，合并后只接受第一个元素，其他元素被简单地忽略。

#4 FlatMapLatest

flatMapLatest 与 flatMap 的唯一区别是：flatMapLatest 只会接收最新的 value 事
件

flatMapLatest 操作符将源 Observable 的每一个元素应用一个转换方法，将他们转换成 Observables。一旦转换出一个新的 Observable，就只发出它的元素，旧的 Observables 的元素将被忽略掉。

#5 Map

通过对每个值应用一个函数来转换Obervable发出的值，然后返回新的Observable

#6 MapWithIndex

mapWithIndex 也只是在 map 基础上添加了索引参数。

#7 Scan

scan将一个输入值作为发出的第一个项，它还将源Observable的元素作为第二项一起反馈回函数，然后通过返回一个新的值作为其第一项发射。

#8 ToArray

将源Observable转换成数组的Observable.

RxSwift Creating

Posted on 2019-06-05 | Edited on 2019-06-09

关于创建Observable

#1 Create

您可以使用create操作符从零开始创建一个可观察对象，传递给这个操作符一个函数，该函数接受observer作为其参数，通过适当地调用observer的onNext、onError和onCompleted方法，将这个函数编写为一个可观察的函数。
需要注意的是一个结构良好的Observable必须调用一次observer的onCompleted或onError方法，并且此后不能尝试调用observer的任何其他方法。

#2 Empty/Never

Empty

创建一个不发出任何项但正常终止的Observable(调用onCompleted)

Never
创建一个不发出任何项且不终止的可观察对象

Empty/Never这些对于测试目的很有用，有时也可以与其他可观测值结合使用，或者作为期望其他可观测值作为参数的操作符的参数。

#3 From

作用将各种其他对象和数据类型转换为可观察的

#4 Interval

创建一个Observable，它发出一个按给定时间间隔间隔的整数序列
Interval操作符返回一个可观察到的值，该值发出一个无限的升序整数序列，在两次发射之间选择一个固定的时间间隔。

#5 Just

创建一个发出特定值的Observable
与From类型，但要注意From会将数组里的元素取出再发送出去，而Just只是简单的将数组或对象作为一个单独值发送出去。

#6 Range

创建一个Observable，该对象发出特定范围的序列整数

#7 Repeat

创建一个Observable，该对象多次发出特定的项

该操作符重复发送一个项。

#8 Timer

创建一个Observable，它在给定的延迟之后发出特定的项

#9 Of

此方法创建一个Observable，该实例具有可变数量的元素。
功能与From相同，From接受数组为元素，Of接受可变数量的元素
