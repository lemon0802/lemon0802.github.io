RxSwift Filtering

Posted on 2019-06-17

#1 Debounce

只有在特定的时间跨度已经过去而没有发出另一个元素，才从在新的Observable中发出一个元素

总结： 相邻两个元素的时间间隔少于 debounce 的时间就会被忽略掉，但会发送最后的元素该操作符确保没有两个元素在小于dueTime的时间内发出。

#2 DistinctUntilChanged

DistinctUntilChanged把相邻两个元素做判断，如果相等就忽略掉后一个的元素。也可以自定义过滤方法。

#3 ElementAt

只发出Observable中的第n(从0开始)个元素。

#4 Filter

Filter操作符只允许能通过提供的判断方法的元素，从而过滤Observable的元素。

#5 First

First操作符只发出这个Observabel对象发出的第一项，如果这个可观察对象完成时没有发出任何东西会返回nil，first会把源Observable转换成Single，Single以后再做介绍。

#6 IgnoreElements

跳过所有元素、完成或错误并在Observable发送完成(或错误)时结束。等价于filter，它总是返回false。

#7 Sample

sample 操作符将不定期的对源 Observable 进行取样操作。通过第二个 Observable 来控制取样时机。一旦第二个 Observable 发出一个元素，就从源 Observable 中取出最后产生的元素。

#8 Single

single 操作符将限制 Observable 只产生一个元素，与first类似。如果 Observable 只有一个元素，它将镜像这个 Observable 。如果 Observable 没有元素或者元素数量大于一，它将产生一个 error 事件。

#9 Skip

skip 操作符可以让你跳过 Observable 中头 n 个元素，只关注后面的元素。

#10 SkipDuration

skipUntil

跳过源 Observable 中头几个元素，直到另一个 Observable 发出一个元素。

skipWhile

skipWhile 操作符可以让你忽略源 Observable 中头几个元素，直到元素的判定为false后，它才镜像源 Observable。

#11 Take

take

通过 take 操作符源 Observable 只发出前 n 个元素。并且忽略掉后面的元素，直接结束序列。

take(duration)

take(duration) 只发出 n 秒前的元素。

takeLast

通过 takeLast 操作符你可以只发出尾部 n 个元素。并且忽略掉前面的元素。

takeUntil

忽略掉在第二个 Observable 产生事件后发出的那部分元素

takeWhile

生成一个 Observable 直到某个元素的判定为 false

#12 Throttle

Throttle 操作符确保没有两个元素在小于dueTime的时间内发出。

如果把 throttle的值变为0.9的话

Throttle和Debounce的区别

Throttle: the original function be called at most once per specified period.
Debounce: the original function be called after the caller stops calling the decorated function after a specified period.
