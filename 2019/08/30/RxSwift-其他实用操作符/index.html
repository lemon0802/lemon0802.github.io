RxSwift 其他实用操作符

Posted on 2019-08-30 | Edited on 2020-02-20

#1 Delay

1.1.delay

将 Observable 延迟一段时间后发出。

总结： 该操作符会将创建一条新的 Observable ，把所有元素都先拖延一段设定好的时间，然后才将它们发送出来。

1.2.delaySubscription

延迟订阅

总结： 使用该操作符可以进行延时订阅。即经过所设定的时间后，才对 Observable 进行订阅操作。

#2 Materialize

2.1 materialize

  • 可把订阅元素具体化成事件。

2.2 dematerialize

dematerialize 与 materialize 是相反的，把事件(Event) 转换成元素(T)。

#3 Timeout

如果在一段特定的时间内没有任何发出的项，则发出onError事件。

#4 Using

创建与 Observable 具有相同寿命的可使用资源。当源Observable 终止时，using 创建的资源也会被销毁。

使用方法

using<Resource: Disposable>(_ resourceFactory: @escaping () throws -> Resource, observableFactory: @escaping (Resource) throws -> Observable<E>) -> Observable<E>

#5 Share

返回一个可观察的序列，该序列与底层序列共享单个订阅，订阅后立即重播buffer中的元素。默认值是0，buffer全部元素。
相当于source.multicast(makeSubject: { Replay.create(bufferSize: replay) }).refCount()

可共享附加作用。

#6 CatchError

从一个错误事件中恢复，将错误事件替换成一个备选序列
